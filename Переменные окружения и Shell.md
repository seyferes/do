
### **Ключевые команды**

#### `export` — Создание переменных окружения

Делает переменную доступной для дочерних процессов.
**Синтаксис:** `export ИМЯ=ЗНАЧЕНИЕ`

**Пример:**

```bash
export DB_HOST="localhost"
export EDITOR="vim"
```

#### `echo` - Вывод значения

**Синтаксис:** `echo $ИМЯ_ПЕРЕМЕННОЙ`

```bash
echo "My editor is: $EDITOR"
```

#### `env` / `printenv` - Просмотр всех переменных

**Синтаксис:** `env`

```bash
env | grep PATH  # Найти только переменную PATH
```

#### `source` - Применение настроек

Читает и выполняет команды из файла в _текущей_ оболочке. Аналог команды `.` (точка).
- **Синтаксис:** `source путь/к/файлу`

```bash
source ~/.bashrc  # Применить изменения конфига без перезагрузки
```

### **Системные переменные**

- `$PATH` — список директорий, где система ищет программы. Разделяется двоеточием `:`.
	- _Пример добавления:_ `export PATH=$PATH:$HOME/my_scripts`
	
- `$HOME` —Путь к домашней директории текущего пользователя.
- `$USER` — Имя текущего пользователя.
- `$SHELL` — уть к используемой оболочке (например, `/bin/bash` или `/usr/bin/zsh`).
- `$PWD` — Текущая рабочая директория.

### **Pro-Tips**

1. **Кавычки:** Используй двойные кавычки `" "`, если внутри есть другие переменные (например, `"$PATH:/new"`). Одинарные кавычки `' '` воспринимают всё буквально как текст.
2. **Naming:** По общепринятому стандарту переменные окружения пишутся **ЗАГЛАВНЫМИ** буквами (`DATABASE_URL`, `API_KEY`).
3. **Безопасность:** Никогда не фиксируй (commit) файлы со "секретами" (паролями) в Git. Используй `.env` файлы, которые добавлены в `.gitignore`.
4. **Паттерн `$VAR`:** Когда присваиваешь новое значение переменной, часто нужно использовать её _старое_ значение. Всегда пиши `$PATH`, если не хочешь стереть все системные пути.
    - _Верно:_ `export PATH=$PATH:/new/dir`
    - _Ошибка:_ `export PATH=/new/dir` (теперь `ls`, `cd` и `grep` не работают).
5. **Секреты:** Никогда не пиши пароли и API-ключи напрямую в `.bashrc`, если ты планируешь показывать свой конфиг (dotfiles) другим. Выноси их в отдельный файл (например, `.env.local`) и делай `source .env.local`.

### **Подводные камни**

- **Забытый export:** Если просто написать `VAR=1`, программа, которую ты запустишь следом, эту переменную не увидит.
- **Ошибка с PATH:** Если ты случайно сделаешь `export PATH=/my/path` (забыв добавить `:$PATH`), то у тебя перестанут работать `ls`, `cd` и все остальные команды, пока ты не исправишь это или не перезайдешь.
- **Одинарные vs Двойные кавычки:**
	- `echo "$USER"` -> выведет `lunnate` (переменная раскрылась).
	- `echo '$USER'` -> выведет `$USER` (текст как есть).
