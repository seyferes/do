
| ***Оператор*** | ***Описание***                                                                                                                                                        |
| -------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `>`            | Перенаправить stdout в файл (**перезаписать**).                                                                                                                       |
| `>>`           | Перенаправить stdout в файл (**дописать** в конец).                                                                                                                   |
| `2>`           | Перенаправить только ошибки (stderr).                                                                                                                                 |
| `&>`           | Перенаправить **всё** (stdout + stderr) в один файл.                                                                                                                  |
| `2>&1`         | Старый синтаксис `&>` направь поток 2 туда же, куда сейчас смотрит поток 1 в старом синтаксисе порядок имеет значение. `> file 2>&1` сработает, а `2>&1 > file` — нет |
| `tee`          | "Тройник": читает из stdin и пишет одновременно в stdout и в файл.                                                                                                    |
| `/dev/null`    | "Черная дыра". Команда `&> /dev/null` заставит любую программу замолчать навсегда.                                                                                    |
#### Pro-Tips:

- **Тихое выполнение:** В Cron-задачах (автоматических скриптах) часто используют `command > /dev/null 2>&1`, чтобы скрипт не спамил в логи, если всё идет хорошо.
- **`tee` под sudo:** Если тебе нужно записать что-то в защищенный файл, `echo "data" > /etc/config` не сработает (права не те). Поможет `echo "data" | sudo tee /etc/config`. Это классический вопрос на собеседовании.

**Подводные камни:**

- **Порядок важен:** `grep "pattern" file > result.txt` — сначала выполняется поиск, потом запись. Если ты попробуешь сделать `grep "pattern" file > file` (запись в тот же файл), ты его **сотрешь** до того, как grep успеет его прочитать.
- **Пайпы и ошибки:** По умолчанию пайп `|` передает только успешный вывод. Если первая команда упала с ошибкой, вторая может получить пустую строку и отработать непредсказуемо.
