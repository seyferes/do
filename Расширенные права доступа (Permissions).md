
### Типы специальных прав


| **Бит** | **Число** | **Символ**  | **На файле**                                  | **На директории**                            |
| ------- | --------- | ----------- | --------------------------------------------- | -------------------------------------------- |
| SUID    | 4         | `s` (user)  | Запуск с правами **владельца** (обычно root). | (Не используется)                            |
| SGID    | 2         | `s` (group) | Запуск с правами **группы**.                  | Новые файлы наследуют **группу** директории. |
| Sticky  | 1         | `t` (other) | (Не используется)                             | Удалять файл может только его **владелец**.  |

### **Синтаксис и примеры (chmod)**

Ты можешь ставить права двумя способами: через числа (абсолютный метод) или через символы (относительный).

#### **Числовой метод (Рекомендуется для скриптов)**

Формат: `chmod ABCD файл`, где A — спецбиты, BCD — стандартные права.

- **SUID:** `chmod 4755 app` (4 = SUID).
- **SGID:** `chmod 2775 project` (2 = SGID).
- **Sticky:** `chmod 1777 public` (1 = Sticky).
- **Комбо:** `chmod 6755 app` (4+2 = SUID + SGID).

#### **Символьный метод (Удобнее для людей)**

- **Добавить SUID:** `chmod u+s file`
- **Добавить SGID:** `chmod g+s dir`
- **Добавить Sticky:** `chmod +t dir`
- **Убрать бит:** `chmod u-s file`

### **Pro-Tips & Security**

1. **Поиск дыр:** Регулярно ищи "забытые" SUID файлы, через которые хакеры могут поднять права: `find / -perm /4000 -type f 2>/dev/null`
2. **Совместная работа:** Всегда используй **SGID** (`chmod 2775`) на папках общих проектов, иначе коллеги не смогут редактировать файлы друг друга.
3. **`chmod 777` — Табу:** Никогда не ставь 777 просто так. Если нужно разрешить запись всем, ставь **1777** (с Sticky Bit), чтобы защитить файлы от случайного удаления соседом.
