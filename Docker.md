
## Команды: Сборка, запуск, логи 

1. `docker build -t my-first-image .` 
	-  **`build`** — Команда превращения **Dockerfile** в **Image (Образ)** 
	- **`-t`** (tag) — Позволяет дать образу имя, если его не указать, образ получит длинный случайный ID
	- **`.`** — В директории, в которой сейчас находимся
2. `docker run -d -p 8081:80 my-website`
	- **`-d`** (detached) — запустить в фоновом режиме.
	- **`-p 8081:80`** — самое важное! Это «проброс портов». Cоединение порта **8081** на  реальном компьютере с портом **80** внутри контейнера.
3. `docker ps` —  основной «радар». Показывает контейнеры, которые работают сейчас.
	- **`-a`** — Показывает все контейнеры и работающие, и остановленные
4. `docker stop [ID или имя]` — Остановить образ (контейнер)
5. `docker rm [ID или имя` — удаляет «труп» остановленного контейнера. Если его не удалить, он будет просто занимать место на диске
6. `docker logs [ID или имя]` — показывает «выхотп» программы внутри контейнера.
	-  **Это первое, что нужно делать, если сайт не открывается.**


- `docker exec -it [имя_контейнера] bash` — Вход внутрь контейнера
	- `-it` — делает сессию интерактивной (можно печатать)
	- `bash` — запускает командную оболочку внутри контейнера

## Docker Volumes: Разработка без границ

В Docker есть механизм **Volumes** (Тома), который позволяет «пробросить» папку с рабочего компьютера прямо внутрь контейнера
Можно создать окно. менять файл на рабочем компьютере, и он **мгновенно** меняется внутри запущенного контейнера.

Команда запуска с томом: 

`docker run -d -p 8085:80 --name workshop-site -v $(pwd):/usr/share/nginx/html nginx:latest`
 -  `-v` — Флаг для тома (volume)
 - `$(pwd)` — команда, которая подставляет путь к текущей папке
 - `:usr/share/nginx/html` — путь внутри контейнера, куда "клеим" нашу папку
